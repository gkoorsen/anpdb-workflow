# Cofactor Parameterization Notes

This repository supports two production MD paths:

- OpenMM ffxml assembly for systems that can be built safely from receptor, ligand, and force-field files.
- Amber-prepared full systems for cofactors that need target-specific chemistry.

Use the Amber-prepared path for CYP1B1 and MAO-B unless you already have validated OpenMM ffxml templates for the exact cofactor states.

## CYP1B1 Heme

The 4I8V structure contains heme coordinated by Cys457 SG to HEM FE. Do not parameterize this as a generic free heme ligand.

The current curated CYP1B1 system is generated by:

```bash
conda run -n Docking python scripts/md_build_amber_systems.py --target cyp1b1 --force
```

The builder uses:

- `data/md_inputs/receptors/4I8V_chainA_heme_prepared.pdb`
- `data/md_inputs/poses/Mol_11315_4I8V_out.pdbqt`
- Amber ff14SB for protein.
- GAFF2 and AM1-BCC charges for Mol_11315.
- AmberTools `dat/contrib/Shahrokh_heme/IC6` P450 heme parameters for ferric penta-coordinate high-spin HEM and the proximal cysteine ligand.
- TIP3P water with an estimated 0.15 M NaCl salt level.

The generated production files are:

```text
data/md_inputs/amber_systems/cyp1b1_mol11315.prmtop
data/md_inputs/amber_systems/cyp1b1_mol11315.inpcrd
```

The generated build manifest and Leap logs are local run artifacts under:

```text
runs/amber_system_prep/cyp1b1_mol11315/
```

After generation, validate with:

```bash
python scripts/md_check_amber_inputs.py
python scripts/md_production_amber.py --config configs/md/amber_production/cyp1b1_mol11315_100ns_rep1.toml --dry-run
```

MCPB.py remains a reasonable alternative if you decide to derive a target-specific heme metal-center model instead of using the curated AmberTools IC6 P450 parameter set. In that case, document the oxidation, spin, and ligation state and replace the generated Amber system files.

## MAO-B FAD

The 2V5Z MAO-B structure contains FAD, and human MAO-B FAD is covalently attached to Cys397. Do not parameterize FAD as a free noncovalent ligand unless the scientific question explicitly calls for an apo or noncovalent control.

The repo does not currently generate a MAO-B Amber system because no curated covalent 8alpha-S-cysteinyl-FAD Amber residue/parameter set is present locally. Generating free-FAD GAFF parameters and then adding a manual SG-C8M bond would use the wrong valence and charge model for publication work.

Required MAO-B input:

```text
Curated Amber-compatible residue/library/frcmod for oxidized FAD covalently linked to Cys397,
or a reviewed CHARMM-GUI/AmberTools output for the full MAO-B/FAD/Mol_14056 system.
```

When that input is available, save the final solvated files as:

```text
data/md_inputs/amber_systems/maob_mol14056.prmtop
data/md_inputs/amber_systems/maob_mol14056.inpcrd
```

Then validate with:

```bash
python scripts/md_check_amber_inputs.py
python scripts/md_production_amber.py --config configs/md/amber_production/maob_mol14056_100ns_rep1.toml --dry-run
```

## Git LFS

Amber topology and coordinate files are intentionally tracked through Git LFS:

```bash
git lfs install
git add data/md_inputs/amber_systems/cyp1b1_mol11315.prmtop data/md_inputs/amber_systems/cyp1b1_mol11315.inpcrd
git add data/md_inputs/amber_systems/maob_mol14056.prmtop data/md_inputs/amber_systems/maob_mol14056.inpcrd
```

## Why Full Amber Systems

OpenMM can read Amber `prmtop` and `inpcrd` directly. For heme iron coordination and covalent FAD, full-system Amber preparation keeps the complex cofactor chemistry in one reviewed topology instead of forcing it into separate ffxml files after the fact. The production runner records the exact topology and coordinate paths in `run_manifest.json`.
